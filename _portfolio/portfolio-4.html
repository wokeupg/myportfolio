Introduction

This project was completed as part of a team of three for our semester-long CS 411: Database Systems course project. I specifically worked on the relational schema, 3NF data normalization, functional dependencies, advanced sql queries, CRUD commands, and the user login interface.

For first-time dog owners, choosing the right breed can be difficult. Our database of over 360 breeds includes temperament, activity, trainability, and energy level. By creating a profile and selecting preferences, users receive tailored breed matches along with nearby shelters and ethical breeders for adoption.

The visualization below represents customer data retrieved from the database that my team and I designed and built. I implemented two advanced queries to generate meaningful insights: one displays breed matches per customer, and the other shows the locations associated with those breeds. These queries help narrow down breed choices according to customer preference regional breed availability.


Query Description and Raw Code

Query 1: BreedMatches per Customer
This query identifies the top 10 dog breeds that are the best fit for a specific customer. It compares each breed’s energy level to the customer’s preferred energy (allowing a difference of ±1), trainability to the customer’s preferred trainability (±2), and checks that at least one of the breed’s temperament traits matches the customer’s desired traits. Breeds are then ranked based on how closely their traits align with the customer’s preferences.

Query 2: ShelterMatching to Customer’s Breed Matches
This query finds shelters that have the breeds selected in Query 1. It ranks the shelters by proximity to the customer: first by the same city, then by the same state, and finally by different states. The query returns the top 5 shelters along with their address details and the dog breeds they offer, helping customers locate compatible dogs nearby.

Query 1: BreedMatches per Customer Query 

SELECT
    d.DogBreedName,
    d.DogBreedID,
    d.Temperament AS BreedTemperament,
    p.CustPrefTemperament AS CustomerPrefs,
    d.EnergyLevelValue AS BreedEnergy,
    p.EnergyLevel AS CustomerEnergyPref,
    d.TrainabilityValue AS BreedTrainability,
    p.CustPrefTrainability AS CustomerTrainabilityPref
FROM DogBreed d
JOIN CustomerPreference p 
    ON d.EnergyLevelValue BETWEEN p.EnergyLevel - 1 AND p.EnergyLevel + 1
   AND d.TrainabilityValue BETWEEN p.CustPrefTrainability - 2 AND p.CustPrefTrainability + 2
   AND (
        d.Temperament LIKE CONCAT('%', TRIM(SUBSTRING_INDEX(p.CustPrefTemperament, ',', 1)), '%')
     OR d.Temperament LIKE CONCAT('%', TRIM(SUBSTRING_INDEX(SUBSTRING_INDEX(p.CustPrefTemperament, ',', 2), ',', -1)), '%')
     OR d.Temperament LIKE CONCAT('%', TRIM(SUBSTRING_INDEX(p.CustPrefTemperament, ',', -1)), '%')
   )
WHERE p.CustomerId = 3
ORDER BY 
    ABS(d.EnergyLevelValue - p.EnergyLevel) ASC,
    ABS(d.TrainabilityValue - p.CustPrefTrainability) ASC
LIMIT 10;
Query 2: ShelterMatching to Customer’s Breed Matches 

WITH BreedMatches AS (
    SELECT
        d.DogBreedID,
        d.DogBreedName,
        d.Temperament AS BreedTemperament,
        p.CustPrefTemperament AS CustomerPrefs,
        d.EnergyLevelValue AS BreedEnergy,
        p.EnergyLevel AS CustomerEnergyPref,
        d.TrainabilityValue AS BreedTrainability,
        p.CustPrefTrainability AS CustomerTrainabilityPref
    FROM DogBreed d
    JOIN CustomerPreference p 
        ON d.EnergyLevelValue BETWEEN p.EnergyLevel - 1 AND p.EnergyLevel + 1
       AND d.TrainabilityValue BETWEEN p.CustPrefTrainability - 2 AND p.CustPrefTrainability + 2
       AND (
            d.Temperament LIKE CONCAT('%', TRIM(SUBSTRING_INDEX(p.CustPrefTemperament, ',', 1)), '%')
         OR d.Temperament LIKE CONCAT('%', TRIM(SUBSTRING_INDEX(SUBSTRING_INDEX(p.CustPrefTemperament, ',', 2), ',', -1)), '%')
         OR d.Temperament LIKE CONCAT('%', TRIM(SUBSTRING_INDEX(p.CustPrefTemperament, ',', -1)), '%')
       )
    WHERE p.CustomerId = 3
    ORDER BY 
        ABS(d.EnergyLevelValue - p.EnergyLevel) ASC,
        ABS(d.TrainabilityValue - p.CustPrefTrainability) ASC
    LIMIT 10
)

SELECT 
    s.ShelterId,
    s.ShelterName,
    s.StreetAddress,
    s.City,
    s.State,
    s.ZipCode,
    b.DogBreedID,
    b.DogBreedName,
    c.City AS CustomerCity,
    c.State AS CustomerState,
    CASE 
        WHEN s.City = c.City THEN 'Same City'
        WHEN s.State = c.State THEN 'Same State'
        ELSE 'Different State'
    END AS Proximity
FROM ShelterProfile s
JOIN ShelterOffering so 
    ON s.ShelterId = so.ShelterID
JOIN BreedMatches b 
    ON so.DogBreedID = b.DogBreedID
JOIN CustomerProfile c 
    ON c.CustomerId = 3
ORDER BY 
    CASE WHEN s.City = c.City THEN 0 ELSE 1 END,
    CASE WHEN s.State = c.State THEN 0 ELSE 1 END,
    s.ShelterName
LIMIT 5;
